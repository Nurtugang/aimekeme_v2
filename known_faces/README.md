# known_faces — мини-база лиц

База лиц живёт **в этом репозитории** (не у платформы: у них — журнал событий, а
эталоны и эмбеддинги привязаны к модели и принадлежат AI-API). Управляется через
**API**, обновляется вживую (без перезапуска сервиса).

Модель распознавания — **ArcFace (`buffalo_l`, insightface)**. Поддерживается
**несколько эталонных фото на человека** — это сильно повышает точность.

Раскладка на диске:

```
known_faces/
├── faces.db        # SQLite: faces(id, name, created_at)
├── 1_1.jpg         # эталон 1 человека id=1
├── 1_2.jpg         # эталон 2 человека id=1
├── 4_1.jpg
└── ...
```

- `faces.db` — источник истины по id/имени; ID автоинкрементны и **не
  переиспользуются** после удаления (на них ссылается журнал событий платформы).
- `<id>_<k>.jpg` — эталонные фото человека (k = 1..N).
- эмбеддинги (512-мерные, L2-нормированы) считаются на старте из фото и живут в
  памяти; в БД их не храним — сменим модель, просто пересчитаем.

## API

- `POST   /faces` — записать человека. `multipart/form-data`: `name` + `images`
  (1..N файлов, на каждом **одно чёткое лицо**). Годные фото эмбеддятся, негодные
  (0 или >1 лица) — отбрасываются. Если ни одного годного → `422`. Вернёт
  `{id, name, created_at, photos}`.
- `POST   /faces/{id}/images` — догрузить ещё фото человеку (другие ракурсы).
- `GET    /faces` — список `[{id, name, created_at, photos}]`.
- `GET    /faces/{id}/image` — первичное (первое) фото человека.
- `DELETE /faces/{id}` — удалить человека (все фото + эмбеддинги).

Ограничения на загрузку (см. `app/config.py`):
- размер одного файла ≤ `FACE_MAX_UPLOAD_BYTES` (5 МБ), иначе `413`;
- большие фото даунскейлятся до `FACE_MAX_IMAGE_SIDE` (1600 px);
- учитывается EXIF-ориентация, фото пересохраняется в JPEG.

Ошибки: нет годных лиц → `422`; имя занято → `409`; нет человека → `404`.

## Как работает узнавание

Входящее лицо сравнивается с эталонами по косинусной близости; берётся
**максимум по всем фото** каждого человека. Если лучший ≥ `FACE_MATCH_THRESHOLD`
(**0.42**) — в ответе `identity` (имя) и `identity_id` (ID), иначе
`identity = "unknown"`, `identity_id = null`. У ArcFace разные люди дают ~0.0–0.3,
один и тот же — ~0.5–0.9, поэтому порог надёжен.
