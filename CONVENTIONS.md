# Соглашения: как добавить алгоритм

Архитектура — **модульный монолит**: один репозиторий, один FastAPI-процесс,
один общий torch. Каждый алгоритм (fight, face, counting, fire, ...) — это
**папка** `app/<name>/` со своим эндпоинтом. Модели грузятся **один раз при
старте** и живут в `app.state.detectors`.

## Раскладка алгоритма

```
app/<name>/
├── model.py      # загрузка модели + (опц.) препроцессинг — детали конкретной сети
├── detector.py   # класс <Name>Detector: load() + predict(); вся ML-логика
├── schemas.py    # Pydantic: запрос и ответ
└── router.py     # APIRouter: POST /detect/<name>
```

Эталоны для копирования: `app/fight/` (вход — клип 16 кадров) и
`app/face/` (вход — 1 кадр, есть мини-база).

## Обязательные правила

1. **Один процесс, общий torch** — никаких отдельных сервисов/портов.
2. **Модель грузится один раз при старте** (в `lifespan` главного `main.py`), не на запрос.
3. **Stateless**: запрос самодостаточен.
4. Детектор реализует интерфейс: `load()`, свойства `is_ready`/`device`, метод `predict(...)`.
5. GPU-инференс под `self._lock` + `torch.inference_mode()` (одна карта на всех).
6. Эндпоинт: `POST /detect/<name>`. Ответ **всегда** содержит `processing_ms`;
   остальные поля — свои у каждого алгоритма (вход/выход у всех разный, это норма).
7. Ошибки клиента — HTTP 422 + `{ "detail": "..." }`.
8. Детектор берётся в роутере через `request.app.state.detectors["<name>"]`.

## Как добавить новый алгоритм (по шагам)

1. Скопировать `app/face/` → `app/<name>/`.
2. Переписать `model.py` (своя сеть) и `schemas.py` (свой вход/выход).
3. Поправить `detector.py` (`predict`) и `router.py` (путь `/detect/<name>`,
   ключ `request.app.state.detectors["<name>"]`).
4. Зарегистрировать в `app/main.py`:
   - в `lifespan`: создать детектор, `.load()`, добавить в `app.state.detectors`;
   - `app.include_router(<name>_router)`.
5. Настройки (пороги и т.п.) — добавить полями в `app/config.py`.
6. Веса не коммитим — пусть качаются в кеш при `load()`.

## Запуск (всё сразу, одна команда)

```bash
uvicorn app:app --host 0.0.0.0 --port 8000   # /docs покажет все /detect/* эндпоинты
```

## Когда отступать от монолита

Если готовая библиотека алгоритма конфликтует по зависимостям с общим стеком
(несовместимая версия torch/onnx и т.п.) — только тогда выносим этот алгоритм
в отдельный процесс/сервис. Пока конфликта нет — всё в одном процессе.
